# 04 | Simon Game (Logic & State)

A browser-based implementation of the classic memory game "Simon". This project is a strict exercise in **Game Loop Logic** and **Asynchronous State Management**.

## 🧠 The Logic Engine

The game relies on two arrays to track state:
1.  `gameSeq`: The master sequence generated by the computer.
2.  `userSeq`: The temporary sequence input by the player.

### Core Algorithms
* **Level Progression:** When `userSeq.length === gameSeq.length`, the game validates the input. If correct, it triggers `levelUp()`, which clears the user array and pushes a new random color to the master sequence.
* **Pattern Matching:** The checking function compares the most recent user click against the corresponding index in the master sequence.
    * *Match:* Continue.
    * *Mismatch:* Trigger "Game Over" state and reset high score logic.

### Visual Feedback
Used `setTimeout` to manage the "flash" effect of buttons, ensuring the visual cue lasts exactly 250ms before reverting, creating a responsive game feel.
